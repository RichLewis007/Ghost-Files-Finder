# This is a rclone filter file, listing rules which control what files/folders rclone should exclude from transferring, listing, checking, etc.
#
# Note: Rclone matching rules follow a glob style, not regex.
#       But Regexp syntax can be mixed with glob syntax. (dangerous to do!)
#       see: https://rclone.org/filtering/
#
# Note: * matches any sequence of non-separator (/) characters
# Exclude common unwanted files/folders at any level

# Using rclone's --filter-from is more flexible than --exclude or --exclude-from and the same filter file is usable with sync, copy, ls, check, etc.

# dir/** is the standard pattern to exclude the contents of a directory recursively.
# Patterns ending with / are treated as directory rules and are used to optimize recursion; if they are excludes, rclone will not recurse into those directories. That effectively drops all their children as well.

# Exclude unwanted files and folders
# Uses - <pattern> rules to exclude unwanted files/folders
# rclone's first-match logic means earlier excludes win unless a higher-priority include is above them.
#
# Important Note: Do NOT put comments on the same line as filter rules! rclone treats these as port of the filter, so the filter will fail.
#
# Do NOT need to end a rclone "filter-from" list with "+ *" to include ALL other files. This is default behavior.
# This list is ready for use with "rclone check" or "rclone sync" like this: "--filter-from [FILTER-NAME]"
#
